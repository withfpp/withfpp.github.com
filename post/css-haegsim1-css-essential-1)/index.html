CSS 핵심1 ( CSS essential 1)

CSS, 핵심을 정리해보자.

block VS Inline

block 요소 : 전체 공간(너비)을 다 차지한다.

div,p,form,section,header,footer...

inline 요소 : 자신의 공간 만큼만 차지한다.

span, a ...

inline 요소에 margin을 넣고 싶다면, inline-block 으로 바꾸어 주어야 한다.

"아니 똑같은 건데 여긴 되는데 여긴 안돼?" 라는 종류의 문제에 닥쳤을 때 많은 경우 블록 요소와 인라인 요소를 구분하지 못해서 생기는 일이 많다는 점을 떠올리면 문제가 의외로 빨리 해결될지도 모른다.

float 을 하면 무너져요

block 속성인 div 안에 자식 요소가 float 속성을 갖게 되면 레이아웃이 망가진다. clear:both 에 대해 반드시 알아두어야 한다.

position 개념

- static
- relative
- absolute
- fixed

기본은 static이다. 아무것도 설정하지 않았다면 static 이다.

relative는 dom이 원래 있던 자리, 그 자리 자체를 기준으로 효과가 적용된다.

absolute 는 가장 가까운 relative 부모를 찾아 참조하게 된다. 그리고 그것을 기준으로 움직이게 된다. 만약 relative 참조 요소가 없다면 browser viewport에 relative 하게 된다. 즉, 브라우저 자체를 참조해서 속성이 적용된다.

absolute 속성을 적용할 때에는 relative 를 어느 부모 요소에 적용할지를 생각해야 한다는 것을 명심하면 이해가 쉽다.

fixed 는 오직 브라우저에 relative 하다는 점에서 absolute와 다르다. 흔히 볼 수 있는 고정되어 있는 네비게이션 혹은 푸터가 fixed 속성을 갖고 있다.

셀렉터를 잘 알면 일이 줄어든다.

css를 이용해서 의외로 많은 코드를 줄일 수 있다는 것을 깨닫고 있다. 가장 기본이되는 것은 셀렉터이다. 셀렉터를 다양하게 활용할 수 있다면 조금 더 단순한 방법으로 문제 해결이 가능하다.
main a      //밑에있는 a 태그 선택
main > a        //main 직계 자식 a 태그
[class]     //클래스를 가진 모든 요소 선택
input[type="text"]      //모든 텍스트 인풋 선택

---pusedo class
a:link
a:visited
a:hover
input:focus
:focus
li:first-child      
li:last-child


--substring matching attribute selector
//$ :  ~로 끝나는
//^ :  ~로 시작하는(caret)

a[href^="http://"]      //http://로 시작하는 즉, 외부링크를 가지고 있는 모든 a 태그를 선택한다.
a[href^="mailto:"]
a[href$=".pdf"]     //.pdf 로 끝나는 링크를 가진 모든 a 태그
a[href$=".doc"]     //.doc 로 끝나는 링크를 가진 모든 a 태그
img[src*="human"]   //이미지 소스 파일에 human 이라는 문자열이 있다면 선택된다. 예를 들어 human-body.png 가 이미지 태그의 소스라면 선택된다.


//li:nth-child()
li:nth-child(2n+4)  //4번째 선택한 뒤 매 2번째마다 선택한다.
li:nth-child(odd)   //매 홀수 li 를 선택한다.
li:nth-child(-n +3) //1 부터 3까지 선택한다. 세번째에서 앞으로 땡겨온다고 생각하면 편리하다.
li:nth-last-child(-n+3) // 마지막에서 3개의 li 태그를 선택한다.


div:nth-of-type(odd)    //매 홀수 div
div:nth-last-of-type(1) //마지막 부터 세어서 첫 번째

:target     //타겟이 링크된 div에 도착하면 적용될 스타일을 지정할 수 있다.
:empty      
div:not(:empty) //비어있지 않은 모든 div 를 선택한다.
div:not()       //부정(!, not)을 이용하는 선택자이다.
div:not([id="nav"]) //id가 nav 이 아닌 div 를 선택한다.

한 번에 다 외울 필요는 없다고 생각한다. 다만, "이런식으로도 선택이 가능하구나!" 라는 것을 알아두면 css를 더욱 다양하게 활용할 수 있다.

input[type="radio"]:checked+label   //체크된 인풋 값의 라벨 태그에 스타일을 지정할 수 있다.

이번에는 psuedo elements 를 보자.

기본적으로 IE7 이하에서는 적용되지 않는다. 그리고 css의 content 프로퍼티를 잘 이용하는 것은 좋은 테크닉이 될 수 있다.

article::first-line     //첫줄에 효과
article::fist-letter    //첫 글자에 효과

.phone::before {
    content:"\2706" //전화번호 앞에 콘텐츠 생성
}

.download::before{
    content: url(../img/pdf.png);
    //다운로드 글자 앞에 pdf 그림을 넣어줄 수 있다.

.download::after{
    content: attr(href);
    //다운로드 클래스에 href 속성을 내용 뒤에 붙여서 표시할 수 있다.
}

background

background-size:cover;  //배경 이미지가 너무 커서 제대로 나오지 않을 경우 div의 크기에 맞게 그림을 나타내준다. height:100% 와 함께 하고 있는지도 확인해야 한다. 

background-attachment: fixed;   //배경 그림은 가만히 있고 글자만 스크롤 시킬 수 있다. 기본은 scroll 로 지정되어 있는데, 그림과 그림이 함께 붙어서 스크롤 된다.

특정 크기에서만 스타일을 적용

미디어쿼리를 이용한다.

// 마크업에는 버튼이 들어가 있다. 모바일 우선 레이아웃이다.
// 그러나, 큰 화면에서는 버튼이 나오게 하고 싶지 않다.
// 전체 브라우저 스크린 ~ 최소 680px 까지는 버튼을 나타내지 말자.

// 679px에 도달하게 된다면 버튼이 다시 나타나게 될 것이다.
// display: none 속성이 더 이상 적용되지 않기 때문이다.

@media screen and (min-width:680px){
    button {
        display: none;
    }
}

자주 만나는 오류의 해결책

1.글자가 줄바꿈 없이 밖으로 나가버린다.

word-wrap:break-word;

2.이미지를 반응형으로 하고 싶은데...

max-width:100%;  //max-width를 이용해보기를 추천한다!

3.margin: 0 auto; 가 안돼요.

블록 요소가 맞는지 확인한다. 그리고 양쪽의 마진이 있으려면 width 가 지정되어 있어야 한다. width가 없으면 양쪽 마진 계산이 불가능하다.
