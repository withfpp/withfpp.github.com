this in JS

이 부분은 http://codeflow.co.kr/question/956/javascript-%EC%9D%98-this-%EB%9E%80-%EB%AC%B4%EC%97%87%EC%9D%B8%EA%B0%80/ 에서 발췌하였다. 매우 좋은 글이므로 원문을 참조하십시오.(저는 개인적으로 쉽게 보기위해 발췌합니다.)

모든 프로그래밍 언어에는, 현재 scope 와 현재 context 라는 개념이 있다. 자바스크립트에는 ==lexical scope== 과 ==this== 라는 context 가 있다.

자바스크립트에서는 모든 새로운 scope 들은 function 의 정의로 인해 생성된다. 하지만 C 같은 언어와는 다르게, 이것이 scope 를 만드는 유일한 방법이다. 반복문도, if 문도, {} 블락 도 scope 를 만들지 않는다. 이런 단순함은 축복임과 동시에 저주다.

핵심

자바스크립트에서 scope 을 이해하려면 다음을 이해해야 한다:

  • 새로운 scope 은 ==function== 을 통해서만 만들 수 있다. for ... in 등 다른 어떤 것도 scope 을 만들지 않는다.
  • ==var== 문장은 변수를 현재의 scope 에 로컬 변수로 선언한다. ==var== 문장을 사용한 이후가 아닌, ==var== 문장을 포함하는 scope 에 선언하는 것이다. 이 로컬 변수는 바깥 scope 의 같은이름 변수를 가린다.
  • ==this== 와 ==arguments== 를 제외한 모든 변수는 lexical scope 을 따른다. 즉 이들 변수의 의미는 코드상의 위치 에 따라 정해진다.
  • ==this== 와 ==arguments== 변수는 모든 중첩 마다 변화한다. 이 변수들을 closure 에 보관하려면, lexical scope 을 따르는 다른 변수를 이용하여 ==this== 나 ==arguments== 들을 참조하여야 한다.
  • ==this== 의 값은 함수가 호출된 방법 에 의해 정해진다. apply 와 call 을 이용하여 이를 조절할 수 있다.

Dog 과 Person 의 greeting 함수는 같은 함수이다. 이 두 함수는 메모리상의 같은 객체를 참조한다. 하지만 어디서 불렸는지에 따라 this 의 값이 변할 수 있다. 기본적으로 함수호출시 . 앞에 있는 객체가, 함수에서 this 가 된다. Dog.greeting() 에서는 this 가 Dog 이고, Person.greeting() 에서는 Person 이다. == . == 앞에 아무것도 없을 때에는 글로벌 객체가 this 가 된다.

var 는 단순히 어떤 변수가 어떤 scope에 속하는지를 나타내는 키워드이다. 예제를 직접 만들어보았다. 이것만 헷갈리지 않는다면 변수로 고생하는 일은 없을것 같다. 호이스팅은 변수 선언을 올릴 뿐, 할당은 호이스팅과 관련이 없다.

@

기본 this 규칙을 먼저 살펴보자. 1. 기본적으로 this는 전역 객체를 참조한다. 2. 메소드 내부의 this는 해당 메소드를 호출한 부모 객체를 참조한다. 3. 생성자 함수 코드 내부의 this는 새로 생성된 객체를 참조한다. 4. call()과 apply() 메소드로 함수를 호출할 때, 함수의 this는 첫 번째 인자로 넘겨받은 객체를 참조한다. 5. 프로토타입 객체 메소드 내부의 this도 해당 메소드를 호출한 부모 객체를 참조한다. 6. JavaScript의 this 키워드는 접근제어자 public 역할을 한다.

이제 this가 길을 잃는 과정을 살펴본다.
